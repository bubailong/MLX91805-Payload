① 系统概述：传感器如何感知载荷？

MLX91805 是一颗安装在轮胎内壁的加速度传感器。当轮胎以某角速度滚动时，传感器绝大多数时间都在做圆周运动，受到强烈的离心加速度。但当传感器随轮胎转到"接地区"（轮胎被地面压平的那段弧）时，轮胎形变会产生一个方向和大小都异常的加速度脉冲。

关键观察：信号在接地区出现明显的下凹（dip），这是因为接地时离心加速度突然被地面反力抵消，Z 轴读数骤降。Tms_DetectPatchesZ() 就是通过检测这个下凹的起止点来确定接地弧的时间范围。

返回的结果结构体

typedef struct {
  uint16_t i_fe; // 接地前沿（falling edge）的采样索引
  uint16_t i_re; // 接地后沿（rising edge）的采样索引
  uint16_t a_patch; // 接触弧信号幅度（ADC量纲）

⚠️

重要：结构体里没有时间字段！函数只返回采样索引（index）。 要得到时间，必须乘以 sampling_period_us。 但计算 CPR 时，两个时间相除，sampling_period_us 会约分掉， 因此 CPR 计算可以完全在索引域内进行，不需要知道采样间隔。

为什么必须 npatches = 2（检测两个接触弧）？

💡

计算 CPR 需要"一圈的时间"（t_period）。 一个接触弧只能告诉你它自己的时长（t_patch），但无法告诉你一圈多长。 必须等到第二个接触弧出现，用两次接地的起始时刻之差，才能得到一整圈的时间。

1

第一个接触弧到来：检测到 i_fe₀（前沿）和 i_re₀（后沿）。记录但不足以计算 CPR。

2

轮胎继续转动：传感器离开接地区，回到离心加速度状态，等待下一圈。

3

第二个接触弧到来：检测到 i_fe₁（前沿）和 i_re₁（后沿）。至此：
• n_period = i_fe₁ − i_fe₀ → 一整圈经过了多少个采样点
• n_patch  = i_re₁ − i_fe₁ → 第二次接地持续了多少个采样点

4

函数返回 DETECT_DONE，调用方计算 CPR。

③ CPR 完整推导过程

第一步：从索引换算为时间

// 采样索引 → 物理时间（单位：μs）
t_patch = (i_re₁ − i_fe₁) × sampling_period_us // 接触弧时长
t_period = (i_fe₁ − i_fe₀) × sampling_period_us // 一圈总时长

第二步：计算 CPR（接触弧比）

CPR = t_patch / t_period
= [(i_re₁ − i_fe₁) × sampling_period_us]
   / [(i_fe₁ − i_fe₀) × sampling_period_us]
= (i_re₁ − i_fe₁) / (i_fe₁ − i_fe₀) // sampling_period 约分，完全在索引域计算！

第三步：整数定点化（避免浮点运算）

MCU 没有 FPU，使用"放大 10000 倍"的定点整数表示（CPR_x10000 = CPR × 10000）：

// 等效于 CPR×10000，用 uint32 运算（n_patch≤180，×10000=1800000，不溢出）
uint16_t n_patch = (i_re₁ − i_fe₁); // 接触弧采样点数
uint16_t n_period = (i_fe₁ − i_fe₀); // 一圈采样点数

uint32_t cpr_x10000 = ((uint32_t)n_patch × 10000u) / (uint32_t)n_period;

// 例：n_patch=23, n_period=290
// cpr_x10000 = 23×10000/290 = 793 → CPR = 7.93%

第四步：CPR 的物理意义

// 接地弧长 L = 2 × √(2Rδ - δ²) ≈ 2√(2Rδ) （δ≪R 时）
// 一圈周长 = 2πR
CPR ≈ L / (2πR) = 2√(2Rδ) / (2πR) = √(2δ/R) / π

// 轮胎挠度 δ 由载荷 F 决定（胎压 P、接触面积 A 共同决定 δ）：
// 在正常工况下 δ ∝ F （近似线性关系）
// 所以 CPR² ∝ F → 或在小形变范围内 CPR ∝ F（一阶近似）
✅

这就是为什么可以用 CPR（%） 与 载荷（kg）的线性关系 来标定的物理依据——在正常行驶载荷范围内，这个线性近似的误差足够小（通常 < 3%）。

④ 为什么不能直接用 t_patch，必须用 CPR？

对比项 直接用 t_patch（接触时长） 用 CPR = t_patch / t_period
同一载荷，速度加倍 t_patch 减半（误判为载荷减半） CPR 不变（比值恒定）
标定条件 必须在固定速度下标定，否则无效 任意速度下标定一次，处处有效
标定公式 F = k·t_patch·v（需要额外测速） F = k·CPR + b（无需速度项）
物理含义 时间量，与速度强耦合 纯几何量（接地弧角/2π），只依赖形变
工程实用性 车速变化时需实时补偿，复杂 一组 k/b 全速度段通用，简单可靠

数值举例说明

场景：轮胎半径 R = 320 mm，某载荷下接地弧角 θ = 28.6°

低速 v = 30 km/h → 转速 n = 4.14 rps → t_period = 242 ms
  t_patch = 28.6/360 × 242 ms = 19.2 ms
  CPR = 28.6/360 = 7.94%

高速 v = 100 km/h → 转速 n = 13.8 rps → t_period = 72.5 ms
  t_patch = 28.6/360 × 72.5 ms = 5.76 ms（↓ 3.3×！）
  CPR = 28.6/360 = 7.94%（完全不变 ✓）

⑤ 从 CPR 到载荷：线性标定的完整推导

模型建立

在正常行驶载荷范围内（空载～满载），CPR 与垂向载荷 F 满足近似线性关系：

F = k · CPR(%) + b

其中：
  F = 轮胎垂向载荷（kg）
  CPR = Contact Patch Ratio（接触弧比，单位 %）
  k = 斜率（kg/%），载荷每增加 1 kg，CPR 增加 1/k %
  b = 截距（kg），CPR=0 时的等效载荷（通常为负值）

两点标定法（求 k 和 b）

选取两种已知载荷 F₁、F₂（例如空载和满载），分别测量稳定的 CPR₁、CPR₂：

联立方程：
  F₁ = k·CPR₁ + b    ...(1)
  F₂ = k·CPR₂ + b    ...(2)

(2)-(1)：
  F₂ - F₁ = k·(CPR₂ - CPR₁)
  k = (F₂ - F₁) / (CPR₂ - CPR₁)

代回 (1)：
  b = F₁ - k·CPR₁

固件中以整数存储（避免浮点）：
  k_x100 = round(k × 100)   // 保留2位小数
  b_kg = round(b)    // 单位 kg，整数

运行时载荷估算

// 每次检测到稳定 CPR_EMA 后，工作固件计算：
CPR_pct = cpr_ema_x10000 / 100.0f; // CPR × 10000 → CPR%
load_kg = (k_x100 / 100.0f) × CPR_pct + b_kg; // 载荷估算值

// 温度补偿（可选，轮胎橡胶热膨胀影响 CPR 约 0.04%/°C）：
cpr_corr = cpr_ema_x10000 - LC_TEMP_COEFF_KT × (T_now - T_ref);
load_kg = (k_x100 / 100.0f) × (cpr_corr / 100.0f) + b_kg;

p_last_indexuint8_t *（静态变量地址）
会话状态
环形缓冲区的写指针。函数每次采样后自动递增此值，调用方只需保证它在两次调用之间不被修改。

为什么必须是静态变量？

函数以分步方式工作：每次调用采集若干点，记录写指针位置，下次调用从上次停止的地方继续。如果放在栈上，每次调用都清零，前后两个接触弧就无法关联到同一次检测会话，导致始终拿不到 t_period。

// ✅ 正确做法：静态变量，在文件作用域声明
static uint8_t s_last_index; // 每次重新开始检测前手动置 0

// 每次新一轮检测前重置：
s_last_index = 0u;
status = Tms_DetectPatchesZ(result, &s_last_index, ...);

// ❌ 错误做法：局部变量（每次调用都清零）
uint8_t last = 0; // 永远是 0，检测无法累积状态

工作原理（环形缓冲区）

data_buffer_depthuint8_t = 180
固定配置
环形缓冲区的深度（采样点数）。必须能容纳"从第一个接触弧前沿到第二个接触弧后沿"的全部采样点。

最小缓冲区深度推导

需要容纳的窗口：
  一圈采样点数 n_period + 第二个接触弧的 n_patch

一圈采样点数：
  n_period = t_period / Ts = (2π × r) / (v × Ts)

低速（30 km/h = 8333 mm/s）时最大：
  r = 316 mm，Ts = 300 µs
  t_period = 2π × 316 / 8.333 = 238 ms = 238000 µs
  n_period = 238000 / 300 ≈ 793 点 ← 超出 180！

结论：data_buffer_depth = 180 对应的是高速（>60 km/h）场景。
低速下轮胎一圈超过 180 个点，但库的环形缓冲区处理了回绕，
只要能在回绕内找到两个下凹，就能正确返回结果。
实用原则：深度设为 180，与官方示例一致。
npatchesuint8_t = 2
固定配置
需要检测的接触弧数量。计算 CPR 必须传 2，这样才能同时得到 t_patch 和 t_period。

为什么必须是 2？

// npatches=1 时：只有 results[0].i_fe / i_re
t_patch = i_re₀ - i_fe₀; // 接触弧时长 ✓
t_period = ??? // 无法得知一圈时间 ✗
CPR = ??? // 无法计算 ✗

// npatches=2 时：有 results[0] 和 results[1]
t_patch = i_re₁ - i_fe₁; // 第二个弧时长 ✓
t_period = i_fe₁ - i_fe₀; // 两次接地起点之差 = 一圈 ✓
CPR = t_patch / t_period; // 可以计算！✓
sampling_period_usuint16_t，单位 μs，范围 93~16383
动态计算
传感器的 Z 轴加速度采样间隔。太长会漏掉接触弧（采样点不够），太短会超出缓冲区（RAM 溢出）。需要根据车速动态计算。

完整推导过程

// 步骤 1：轮胎挠度 δ（mm）
δ ≈ H × 20% = 截面高度 × 0.20
= (W × AR / 100) × 0.20
例（205/60R16）：δ = (205 × 60/100) × 0.20 = 123 × 0.20 = 24.6 mm

// 步骤 2：接地弧长 L（mm）
L = 2 × √(2Rδ) （δ≪R 近似，见 tire_config.c 精确推导）
例：L = 2 × √(2 × 316 × 24.6) = 2 × √15547 ≈ 2 × 124.7 ≈ 249 mm

// 步骤 3：接地时间 t_patch（μs）
t_patch = L / v = L_mm × 3600 / v_kmh × 1000 （单位换算）
例（80 km/h）：t_patch = 249 × 3600000 / 80000 ≈ 11.2 ms

// 步骤 4：最优采样周期 Ts（μs）
Ts = t_patch / target_samples
例（target=30）：Ts = 11200 / 30 ≈ 373 μs

// 步骤 5：硬件限幅
Ts = clamp(Ts, 93, 16383) （最小 93μs = 硬件最小采样间隔）
